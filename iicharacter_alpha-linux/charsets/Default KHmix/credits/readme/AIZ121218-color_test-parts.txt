パーツ作成支援　色テスト用パーツ

　色を変化させるパーツ作成の際に，どんな色で塗ったら良いか頭を悩ませていませんか？

　髪の色のようなデフォルトパーツと同色を使えば良い（あるいは使わないといけない）ものは，デフォルトパーツからその都度スポイトで掬えばなんとかなりますが，服のようにデフォルトパーツとは違う色合いを出したい時には悩みどころ。
　私の場合，その都度テストしながら色調整をかけて…なんてやっているうちに調整前の元の色が分からなくなってしまって，また最初からやり直し，なんて事を何度もやっています。
　デフォルトパーツから色を掬うにしても，主線の色などは掬う場所によって微妙に色が異なるので，必ずしも同じ色になるとは限りませんし，色変化をさせない色でもグラフィックツールのパレットにないものはやはりどこかに残しておかないと同じ色が使えなくなる事が多々ありますしね。
　そこで以前は掬った色で乱雑に太線を引いたものを色見本用ファイルとして保存してパレット代わりにして同じ色を使えるようにしていました。

　そこから発展させて，パレット兼用としても使えるように色テスト用の枠パーツを作成して，現在ではそれを使用しています。

【パーツ内容】
※基本色変化させたい色のテストを想定していますが，もちろん色変化させない場合のパレットとしてもお使いください。
・color_test-frame：明暗を４段階だと想定して枠４つの左と上にコメント欄を設けたもの。
・color_test-frame2：上にコメント欄を設けた枠５つを２組並べて細かいグラデーション設定にも対応させられるようにしたもの。
・color_test-base：color_test-frameの枠の上の欄に基本色，主線，暗色，明色の文字を書込んで，規格の統一を図ったもの。

※以下３つはcolor_test-baseを用いたサンプル。色変化させるパーツの試験用として作成しています。
・color_test-sample001(服)：名前の通りサンプルデータです。一番上はトライアルした後，何が気に入らなかったのか使用した記憶がありません。２番目は淡い色のトライアル，３番目は対照色のトライアルでそれぞれ実際に使用，４番目はデフォルトのセーラー服からスポイトで掬ったものです。
・color_test-sample002(髪)：サンプルデータその２。デフォルトの髪色を掬ったものです。
・color_test-sample003(服)：サンプルデータその３。上がデフォルトのブレザーから掬った色，２番目が鮮やかな色のトライアルで実際に使用したものです。

【使用方法】
　color_test-frameかcolor_test-baseを開いて各枠内にテスト用の色で塗り潰したりメモを書込んだりした後に，適当にリネームして（パーツリストの上の方に来るように「color_test-番号」辺りが便利？）使用してください。

　パーツ作成支援用のパーツなのでわざわざ書く必要もないかと思いますが，フォルダ分けしていませんので，テスト用のパーツを作成した後，ファイル名の後ろに(髪)，(肌)，(服)の文字列を追加して作成したpngファイルを「accessory_front」と「accessory_back」にコピーしてください。最前面が塗り潰されてしまうパーツなので，「accessory_back」のファイルを空データにする必要はありませんが，その辺はお好みで。

【作成に使用したツール】
・AutoCAD 2006LT
・DocuWorks 6.2
・pixia ver.4.2b
　AutoCADにてベース枠を作成してからA5サイズでDocuWorksファイルに出力後，DocuWorksからPNG出力。Pixiaにてトリミングとリサイズ。
　サンプル作成はpixa。
